# 設計: AI活用実績ポートフォリオサイト

## 1. 目的・想定読者

業務でのAI(Claude Code)活用実績を、社内関係者・取引先・上司にアピールするための単一ページサイト。「業務アプリ・成果物」と「Claude Code 開発ノウハウ」の両系統を1つのサイトに統合し、AI活用の幅広さを示す。

素材: `C:\Users\norih\Desktop\★画像★\今までの成果\` 配下の10枚のスクリーンショット。

## 2. 全体構成(縦1ページ構成)

```
①ヘッダー: 名前(ばたやん) / 肩書きキャッチ(例:「Claude Codeで業務AI活用」)
②ヒーロー: ベントーグリッド(代表作5点、大小不揃い)
   ANA財務ダッシュボード(大) / MBTI64診断ツール / テトリス / ホームページ作成(LP) / MBTI64チームビルダー
③本体: タブ絞り込み一覧(全10点)
   タブ: [すべて] [業務アプリ・成果物(6)] [Claude Code 開発ノウハウ(4)]
   均一グリッドでカード表示(サムネイル+タイトル+一言)
④フッター: 連絡先(任意) / GitHubリンク等
```

②のヒーロー5点は③の一覧にも重複して含める(タブ「すべて」で見たとき10点が一望できる)。

## 3. インタラクション

- ②③のどのカードもクリックすると**モーダル(拡大画像+説明文)**を表示。ページ遷移なし、SPA的に1ページで完結。
- モーダル内容: 拡大スクリーンショット、タイトル、説明文、カテゴリタグ。
- タブ切り替えはクライアントサイドのフィルタリングのみ(再読み込みなし)。

## 4. データ構造

`src/data/works.ts` に作品データを配列で集約(ANA財務ダッシュボードの `financials.ts` と同じ「出典コメント付きで一元管理」方針を踏襲)。

```ts
type WorkCategory = "business" | "knowhow";

interface Work {
  id: string;
  title: string;
  category: WorkCategory;
  isHero: boolean;       // ヒーロー5点フラグ
  thumbnail: string;     // 画像パス
  description: string;   // モーダル内の説明文
}
```

画像は `今までの成果/` の10枚を `src/assets/works/` にコピーして使用。

## 5. 技術構成・公開

- React + TypeScript + Vite(ANA財務ダッシュボードと同構成)
- スタイリング: CSS Modules または軽量な自前CSS(ベントーグリッドは `grid-template-columns/rows` で実装、追加ライブラリ不要)
- 公開: `npm run deploy`(`gh-pages -d dist`)でGitHub Pages。新規リポジトリを作成(例: `norihikokawabata-oss/ai-portfolio-site`)

### 配色・トンマナ(確定)

[`docs/design-tokens/corporate-blue-clean.yaml`](../design-tokens/corporate-blue-clean.yaml)(`corporate-blue-clean-pastel`)を採用。
業務改善レポート風スライドの配色を抽象化したパステル系トーン。

- 背景: `#FAFBFF`(淡い青白) / カード背景: `#F0F4FC`
- キーカラー: `#7C93D9`(パステルネイビー) / アクセント: `#93C5FD`(パステルブルー)
- 強調テキスト色: `#1F2A44` 〜 `#283447`(彩度の高い背景でも視認性を保つため濃紺文字を使用)
- 意味色: 改善・好調 `#86C99B` / 課題・低下 `#F0A8A8`
- 角丸: medium、シャドウ: subtle、区切り線: hairline

CSS変数として `src/styles/tokens.css` 等に上記YAMLの値をそのまま定義し、コンポーネントから参照する。
